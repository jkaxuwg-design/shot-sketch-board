# IP Profile Rules

This skill supports multiple video IPs. Never let one IP's finished-frame examples become the default style for every project.

## Two-Layer System

1. **Universal shot logic**
   - information role
   - viewer takeaway
   - visual metaphor
   - material type
   - 16:9 composition
   - Start/Motion/End frames
   - risk controls

2. **IP-specific visual profile**
   - background style
   - frame shape and stroke
   - presenter position
   - typography mood in final edit
   - material-window ratio
   - highlight color
   - transition rhythm
   - recurring layout patterns

## Current Temporary IP Profiles

### IP-A: Dark Grid Knowledge Explainer

Use only when the user identifies this IP or provides visually matching references.

- Background: dark grid board.
- Frames: white-stroke rounded rectangles.
- Common layouts:
  - left presenter panel + right question wall
  - foreground result panel + background code/process panel
  - real screenshot panel with large edited text outside the panel
- Mood: clear, sharp, knowledge-explainer, not decorative.
- Sketch rule: generated sketch images contain no readable text; finished edit can add large Chinese display text later.

## Adding New IPs

When the user provides new finished-frame examples:

1. Identify whether they belong to an existing IP or a new IP.
2. Extract only reusable visual grammar, not one-off content.
3. Record:
   - layout pattern
   - background
   - frame treatment
   - text placement
   - motion rhythm
   - what script intent triggers this pattern
4. Keep the profile separate from universal rules.
5. Do not overwrite another IP's style.

## Hard Fail

Fail the plan if it blindly applies IP-A dark-grid layout to an unrelated IP without user confirmation.
